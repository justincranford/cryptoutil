## Overview

V13 wires the telemetry pipeline and public-facing endpoints with full TLS/mTLS:

1. **OTel Collector mTLS** — OTLP receiver (gRPC :4317 + HTTP :4318) requires client certs from app instances
2. **Grafana LGTM HTTPS** — UI (:3000) served via HTTPS; OTLP ingest (:14317/:14318) requires client cert from OTel Collector
3. **Public PS-ID App Server TLS** — each app instance serves its public API (:8080) with a dedicated server cert (Cat 3)

V13 depends on V12 Phase 0 being complete (Cat 9 infra cert generated by pki-init).

## Executive Decisions

### Decision 1 (D1): Grafana HTTPS Configuration Approach

**Options**:
- A: Custom `grafana.ini` mounted as volume at `/etc/grafana/grafana.ini`
- B: Environment variables `GF_SERVER_PROTOCOL=https`, `GF_SERVER_CERT_FILE=...`
- C: Docker secret injection for Grafana TLS config
- D: Grafana reverse proxy (nginx/caddy sidecar) — Grafana serves HTTP internally
- E:

**Decision**: Option A selected — custom `grafana.ini`

**Rationale**: grafana.ini provides the most explicit, auditable, and portable configuration.
Environment variables (Option B) conflict with the project policy against inline env vars (only
Docker secrets are permitted for credentials — TLS paths are not credentials, but consistency
favors the config file approach). Option D adds unnecessary complexity.

**Impact**: `deployments/shared-telemetry/grafana-otel-lgtm/grafana.ini` created; compose.yml
mounts it as read-only volume. `__PS_ID__` placeholders used for cert file paths.

### Decision 6 (D6): Grafana OTLP Ingest mTLS Support

**Options**:
- A: Assume grafana/otel-lgtm image supports TLS on OTLP ingest ports (proceed without pre-verification)
- B: Verify support before implementation (spin up container and probe TLS config options)
- C: Use a custom otel-collector-contrib sidecar to terminate TLS before forwarding to Grafana on plaintext
- D: Grafana OTLP ingest over plaintext only (no mTLS for ingest, only UI HTTPS)
- E:

**Decision**: Option A selected — mTLS assumed, verified in Phase 4

**Rationale**: grafana/otel-lgtm embeds OTel Collector for ingest. The embedded collector follows
the same YAML config approach as standalone otel-collector-contrib. If Phase 4 reveals a limitation,
Option C (sidecar) or Option D (plaintext ingest) can be substituted without replanning V13.

**Impact**: Phase 4 includes a verification step for OTLP ingest TLS before proceeding.
